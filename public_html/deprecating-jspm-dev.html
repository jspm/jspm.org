Deprecating jspm.dev after 10 years

Guy Bedford, November 22nd 2023

Over 10 years ago, I made the first release of dev.jspm.io, based on the idea that native modules were coming to the browser and it should be possible to dynamically load modules directly into the browser console.

From the start, there were two CDNs - one version that would just import any package in the browser directly (`dev.jspm.io``) and one made specifically to work with import maps. At the time we called this map configuration, first created in the RequireJS module loader, and then adopted later by the SystemJS loader.

I ended up working on the project as my main open source endeavor while working small contracts on the side, and the project was steadily upgraded with the times. From RequireJS to SystemJS to SystemJS as a polyfill for native modules, to ES Module Shims as a polyfill for native modules, and finally today fully on top of native modules. Following this path was very painful. I have counted that I have rebuilt the CDN architecture itself over 12 times over the years. The CLI has been through 4 different incantations at this point. In the process it alienated a lot of users, not having a clear path. But the path was always clear in my mind to where native modules would land, even if the community became fragmented because of it.
